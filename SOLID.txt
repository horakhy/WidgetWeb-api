# SOLID

1. Single Responsibility Principle (SRP)
2. Open/Closed Principle (OCP)
3. Liskov Substitution Principle (LSP)
4. Interface Segregation Principle (ISP)
5. Dependency Inversion Principle (DIP)

1) Cada classe possui apenas UMA responsabilidade.
2) Cada classe deve ser aberta para extensão e fechada para modificação.
3) Classes devem ser substituíveis por uma classe pai sem grandes problemas (Herança).
4) Classes devem ser independentes, segregar interfaces em funcionalidade.
5) Classes deve estar pronta para receber parâmetros de suas dependências.
